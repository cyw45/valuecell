# 纸面交易资金与收益说明

## 设置初始资金

在“策略”页面的“风险限制”区域填写 **Initial paper capital**，单位为 USDT。

保存新策略时，该金额创建为该策略独立的纸面账户初始现金。策略之间不共享余额，也不会读取交易所账户或真实钱包。

已存在的旧策略没有初始资金记录时，系统会按 10,000 USDT 展示；重新保存策略可明确保留新的本金配置。初始资金只应在尚无成交记录时调整，避免改变历史收益基准。

## 资金何时变化

策略必须处于运行状态，且收到真实 K 线与价格后才会评估。

- `buy`：从纸面现金扣除成交金额，新增纸面持仓。
- `sell`：关闭对应纸面持仓，成交金额回到纸面现金，差额计为已实现盈亏。
- 持仓期间：价格变化计入未实现盈亏和账户权益。
- `no_op`：不成交，账户余额不变化。

每次评估后的账户快照都会持久化，因此刷新页面、重启服务后仍可查看账户余额、持仓、已实现盈亏、未实现盈亏和权益。

## 从保存到收益的操作路径

1. 在“策略”页面填写初始纸面资金并保存。
2. 点击“开始模拟评估”。策略状态必须显示为 `running`。
3. 后端按策略配置的 K 线周期评估。例如 `1h` 策略最多每小时执行一次；它只使用已收盘的真实 K 线。
4. 当规则产生 `buy` 或 `sell` 时，后端记入 `paper filled` 成交并更新纸面账户。
5. 在以下页面查看结果：

| 页面 | 观察内容 |
| --- | --- |
| 策略 | Cash、Equity、已实现/未实现 PnL，及当前策略状态 |
| Dashboard | 初始资金、现金、权益、总 PnL、权益曲线、最近条件 |
| 交易 | 每笔纸面成交的市场、成交价、金额、已实现 PnL、触发原因 |
| 资金 | 以 `当前权益 - 初始资金` 计算的收益曲线 |

刚保存但未启动/未触发成交时，`Cash = Equity = 初始资金`、PnL 为 0，这是正常状态；不是策略已盈利或亏损。

## 市场数据短暂不可用

图表优先使用 Gate，并依次回退到 OKX、Binance、MEXC。若所有实时请求短暂失败，
系统只会保留此前从交易所成功取得的真实 K 线，并将其标记为 `stale`；从未取得过
真实数据时仍显示不可用，不会生成模拟图表。

当外部网络恢复，下一次成功抓取会覆盖旧 K 线。市场数据不可用时，自动策略不会
用缺失价格作出纸面成交。

## 本机启动与端口冲突

本机只启动一个后端实例：

```bash
cd python
uv run python -m valuecell.server.main
```

若出现 `WinError 10048`，表示已有后端正在使用 `127.0.0.1:8000`。不要再启动第二个
实例；先访问 `http://localhost:8000/api/v1/healthz`。返回 200 即说明现有后端可用。

## 查看收益

- **策略页**：保存或评估后，在右侧 `Paper account` 查看现金、权益、已实现/未实现盈亏。
- **Dashboard**：PnL 曲线显示 `当前权益 - 初始资金`。
- **Funding 页面**：同一条权益收益曲线，用于观察策略整体结果。
- **Trades 页面**：仅展示真实纸面成交，`paper filled` 表示账本已更新；不再将未成交推荐写成交易。

## 数据不足时

收益不是回测结果，也不会由前端伪造。如果市场数据提供方不可用、K 线不足，或规则未触发成交，则不会产生可用于判断收益的纸面交易记录。此时应先检查市场数据健康状态，再判断策略表现。

## 纸面模式边界

该功能只记录虚拟 USDT、虚拟持仓与虚拟收益：

- 不发送交易所订单。
- 不需要交易所 API Key。
- 不读取真实账户余额。
- 不涉及真实资金或钱包签名。
