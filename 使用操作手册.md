# ValueCell 使用操作手册

这是一份给日常使用者看的说明，不要求懂代码。

系统的主要用途是：看加密货币行情、配置固定规则策略、先做纸面模拟，再做交易所测试网验证；实盘交易默认关闭，只有完成全部安全步骤后才可能开启。

> 不要把数据库密码、交易所 API Secret、钱包私钥、助记词发到聊天、截图或 Git 仓库。
>
> 本系统不需要、也不接受钱包私钥或助记词。Binance 和 OKX 使用交易所 API Key/Secret；OKX 还需要 Passphrase。

---

## 一、先理解三个交易模式

### 1. 纸面策略（建议第一步使用）

这是系统内部的模拟。

```text
市场行情 → 规则判断 → 买卖信号 → 模拟日志
```

- 不读取交易所账户余额。
- 不会向交易所下单。
- 不会动真实资金。
- 适合先验证策略逻辑是不是符合预期。

### 2. 测试网 / Demo

这是接入交易所的模拟环境。

| 平台 | 测试环境 |
|---|---|
| Binance | Binance Testnet |
| OKX | OKX Demo Trading |

- 使用交易所测试网 API Key。
- 可以读取测试网余额。
- 可以提交测试网订单。
- 测试网里的币是模拟资产，不影响真实账户资金。

### 3. 受控实盘

这是生产交易所真实资金。

- 默认关闭。
- 即使配置了 API Key，也不会自动开启。
- 需要服务器开关、风险策略、策略绑定和本次启动后的人工确认同时满足。
- 请先完整跑通纸面策略和测试网，再考虑实盘。

## 二、本机启动

### 一键启动

在 Windows PowerShell 中执行：

```powershell
cd D:\yolo\valuecell
.\start.ps1
```

它会启动：

1. 前端页面。
2. Python 后端 API。
3. 数据库表初始化。

首次启动会下载/安装依赖，时间会稍长。

### 浏览器地址

通常前端会显示启动地址。后端默认健康检查地址是：

```text
http://localhost:8000/api/v1/healthz
```

如果打开后能看到类似“Welcome to ValueCell”，说明后端正常。

### 停止服务

在启动脚本所在的终端按：

```text
Ctrl + C
```

## 三、数据库在哪里

### 本机运行时配置

本机服务读取：

```text
%APPDATA%\ValueCell\.env
```

通常就是：

```text
C:\Users\Administrator\AppData\Roaming\ValueCell\.env
```

这里应该配置：

```env
VALUECELL_DATABASE_URL=postgresql+psycopg2://用户名:密码@数据库地址:5432/数据库名
```

当前已将运行时目标切到远程 PostgreSQL。重启后端后，新注册账号、策略、订单和交易所连接都会写入远程数据库。

### 如何确认是否连接 PostgreSQL

在 `python` 目录运行：

```powershell
cd D:\yolo\valuecell\python
uv run python -c "from valuecell.server.config.settings import get_settings; print(get_settings().DATABASE_URL)"
```

如果输出以：

```text
postgresql+psycopg2://
```

开头，说明在使用 PostgreSQL。

如果输出以：

```text
sqlite:///
```

开头，说明仍在使用本地 SQLite，需要检查 `.env` 的 `VALUECELL_DATABASE_URL`。

## 四、注册和登录

1. 打开前端页面。
2. 点击“创建账户”。
3. 填写：
   - 工作区名称：随便起，例如“我的量化工作区”。
   - 邮箱。
   - 密码：至少 12 位。
4. 创建成功后会自动进入仪表盘。

每个账号都有独立工作区。你的策略、测试网连接、实盘连接和订单记录都按工作区隔离。

## 五、如何看行情和趋势图

### 仪表盘

仪表盘主要看：

- BTC、ETH、SOL 等行情背景。
- K 线图。
- 累计盈亏曲线。
- 最近策略条件：什么条件触发、什么条件阻止。

### 图表分析

左侧点击“图表”。可以：

1. 选择 BTC、ETH、SOL。
2. 选择 15 分钟、1 小时、4 小时、日线。
3. 在 TradingView 图表中自行添加 MA、RSI、布林带、MACD 等指标。

### 看不到趋势图怎么办

先检查后端行情接口：

```text
http://localhost:8000/api/v1/crypto-market/health
```

重点看 provider 状态：

- `ready`：可用。
- `cooldown`：刚刚请求失败，系统正在等待后重试。

当前默认优先使用 Gate：

```text
Gate → OKX → Binance → MEXC
```

这是因为部分本机网络环境下 OKX、Binance、MEXC 可能超时，而 Gate 公共 K 线通常可用。

行情刷新配置在：

```text
%APPDATA%\ValueCell\.env
```

默认配置：

```env
VALUECELL_MARKET_DATA_PROVIDERS=gate,okx,binance,mexc
VALUECELL_MARKET_DEFAULT_SYMBOLS=BTC-USDT,ETH-USDT,SOL-USDT
VALUECELL_MARKET_DEFAULT_INTERVAL=1h
VALUECELL_MARKET_REFRESH_S=3600
```

含义：服务启动后独立维护 BTC、ETH、SOL 的 1 小时 K 线，默认每小时刷新一次。这个行情缓存和是否启动策略没有关系。

改完 `.env` 后，必须重启后端。

## 六、配置一条纸面策略

左侧点击“策略”。

### 推荐第一次的默认设置

| 项目 | 建议值 |
|---|---|
| 交易对 | BTC/USDT、ETH/USDT |
| K 线周期 | 1h |
| 快速 MA | 20 |
| 慢速 MA | 50 |
| RSI | 开启，超卖 30，超买 70 |
| 布林带 | 开启，周期 20，标准差 2 |
| MACD | 开启，12 / 26 / 9 |
| 仓位大小 | 先用 5% |
| 止盈 | 4% |
| 止损 | 2% |
| 最大持仓数 | 2 |
| 杠杆 | 1x |

### 操作顺序

1. 选择交易对。
2. 选择 K 线周期。
3. 按需要调整 MA、RSI、布林带、MACD。
4. 设置仓位、止盈止损、最大持仓和杠杆。
5. 点击“保存模拟规则”。
6. 看到“保存成功”提示后，点击“开始模拟评估”。

保存失败时，页面应显示具体错误，不再静默失败。

### 自动评估频率

如果策略 K 线周期是：

| K 线周期 | 策略最大评估频率 |
|---|---|
| 5m | 每 5 分钟 |
| 15m | 每 15 分钟 |
| 1h | 每小时 |
| 4h | 每 4 小时 |
| 1d | 每天 |

系统只在完整 K 线周期上评估，避免同一根 K 线重复下单或重复记录。

## 七、怎么看策略是否工作

### 仪表盘的“最新规则条件”

这里会说明：

- 哪些条件已触发。
- 哪些条件未触发。
- 哪些条件因风险限制而被阻止。

例如：

```text
MA 金叉已触发
RSI = 75，高于超买阈值 70
所以不买入
```

### 交易记录

左侧点击“交易”。这里显示：

- 评估时间。
- 建议买入还是卖出。
- 触发原因。
- 数量。
- 执行状态。

纸面策略中的记录只是系统内部建议，不是交易所订单。

### 资金日志

左侧点击“资金”。这里看：

- 模拟累计盈亏曲线。
- 每次评估的资金影响。
- 模拟资金费影响。

## 八、如何测试 Binance Testnet / OKX Demo

进入：

```text
设置 → 测试网交易所
```

### Binance Testnet

准备：

- Binance Testnet API Key。
- Binance Testnet Secret。

### OKX Demo

准备：

- OKX Demo API Key。
- OKX Demo Secret。
- OKX Passphrase。

### 操作顺序

1. 选择交易所。
2. 填写连接名称，例如“Binance 测试网”。
3. 输入测试网 API Key 和 Secret。
4. OKX 额外输入 Passphrase。
5. 保存连接。
6. 点击读取余额。
7. 余额正常后，再提交很小额度的测试网订单。
8. 检查订单状态。

测试网凭据保存时会加密；页面不会回显 Secret。

不要输入：

- 钱包私钥。
- 助记词。
- 钱包签名。
- 真实生产 API Key。

## 九、受控实盘执行：什么时候能开启

进入：

```text
设置 → 受控实盘执行
```

实盘功能默认关闭。页面显示“关闭”是正常状态，不是错误。

### 开启前必须完成

1. 配置生产交易所连接。
   - Binance 或 OKX。
   - 选择现货或合约。
   - API Key、Secret；OKX 还要 Passphrase。
2. 确认交易所 API Key：
   - 已禁用提现权限。
   - 已配置服务器 IP 白名单。
3. 保存实盘风控策略：
   - 单笔最大金额。
   - 最大总敞口。
   - 最大日损失。
   - 最大持仓数。
   - 最大杠杆。
   - 允许交易对。
4. 将“规则策略”绑定到“实盘连接”。
5. 在服务器环境变量中显式设置：

   ```env
   VALUECELL_LIVE_TRADING_ENABLED=true
   ```

6. 重启服务。
7. 每次重启后，在页面请求挑战码，再手动输入确认。

### 为什么每次重启还要授权

这是防止服务器重启后自动恢复真实下单。

实盘授权只保存在当前进程内，默认有效期：

```env
VALUECELL_LIVE_AUTHORIZATION_TTL_S=900
```

即 15 分钟。到期或重启后都需要重新授权。

### 自动实盘下单前会检查什么

每个策略信号都会检查：

```text
全局实盘开关
人工授权是否有效
策略是否绑定实盘连接
交易对是否在白名单
单笔金额是否超限
总敞口是否超限
日损失是否超限
最大持仓数是否超限
余额是否足够
现货/合约杠杆是否合法
同一根 K 线是否已经执行过
```

任意一项不通过：

```text
不下单
只在策略日志里记录“已阻止”和原因
```

## 十、部署到 VPS

### 创建 VPS 运行时环境文件

在 VPS 项目目录中：

```bash
mkdir -p docker/runtime
cp docker/runtime/.env.example docker/runtime/.env
chmod 600 docker/runtime/.env
nano docker/runtime/.env
```

至少填写：

```env
APP_ENVIRONMENT=production
API_HOST=0.0.0.0
API_PORT=8000
API_DEBUG=false

CORS_ORIGINS=https://你的域名

VALUECELL_DATABASE_URL=postgresql+psycopg2://用户名:密码@数据库地址:5432/数据库名
VALUECELL_JWT_SECRET=随机高熵字符串
VALUECELL_CREDENTIAL_MASTER_KEY=32字节base64url密钥

VALUECELL_MARKET_DATA_PROVIDERS=gate,okx,binance,mexc
VALUECELL_LIVE_TRADING_ENABLED=false
```

### 启动 Docker

```bash
docker build -f docker/DockerFile -t valuecell:latest .

docker run -d \
  --name valuecell \
  --restart unless-stopped \
  --env-file docker/runtime/.env \
  -p 8000:8000 \
  valuecell:latest
```

### VPS 网络检查

在 VPS 上执行：

```bash
curl -I --max-time 10 https://api.gateio.ws/api/v4/spot/currencies
curl -I --max-time 10 https://www.okx.com/api/v5/public/instruments?instType=SPOT
curl -I --max-time 10 https://api.binance.com/api/v3/exchangeInfo
```

注意：你电脑上的 VPN 不会影响 VPS。VPS 是否能访问交易所，必须在 VPS 上单独检查。

## 十一、常见问题

### 页面提示“无法加载可用市场”

1. 检查后端是否启动。
2. 打开：

   ```text
   http://localhost:8000/api/v1/healthz
   ```

3. 检查行情 provider：

   ```text
   http://localhost:8000/api/v1/crypto-market/health
   ```

4. 重启后端，让 Gate 优先配置生效。

### 保存策略没有反应

现在保存成功/失败都会有提示。如果仍失败：

1. 确认已登录。
2. 至少选择一个交易对。
3. 快速 MA 必须小于慢速 MA。
4. 止损必须小于止盈。
5. 打开浏览器开发者工具或后端日志，查看具体接口错误。

### 登录后又跳回登录页

1. 确认后端地址是正确的。
2. 刷新一次页面。
3. 清除浏览器中 ValueCell 的本地存储后重新登录。
4. 确认 JWT Secret 没有在服务运行期间变更。

### 想先关闭实盘

保持：

```env
VALUECELL_LIVE_TRADING_ENABLED=false
```

然后重启后端。即使数据库里保存了生产 API 凭据，也不会提交实盘订单。

## 十二、推荐使用顺序

```text
1. 确认行情正常
2. 配置纸面策略
3. 看交易记录和资金日志
4. 测试 Binance Testnet / OKX Demo
5. 验证余额、下单、订单状态
6. 配置生产 API Key（禁提现 + IP 白名单）
7. 配置实盘风控策略
8. 绑定策略和实盘连接
9. 默认继续关闭实盘
10. 最后才显式开启实盘并人工授权
```

不要跳过测试网步骤。实盘出问题时，先关闭全局开关、撤销授权、撤销策略绑定，再检查日志。
